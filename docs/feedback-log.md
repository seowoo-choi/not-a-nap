# NOT A NAP — 피드백 로그 (언니 + 육아서적)

수집일: 2026-07-18. 출처: 언니(실제 부모) 카톡 + 육아서 캡쳐 3장(《0~3세 실전 애착 육아법》 추정).
이 문서는 **백로그**다. 채택 여부는 아래 "스코프 결정" 이후 확정한다. 임의 반영 금지.

## 0. 가장 근본적인 피드백 (게임 구조를 바꾸는 것)

| # | 피드백 | 의미 | 현재 설계와의 관계 |
|---|--------|------|--------------------|
| G1 | **"재우면 그날 성공?" → 아니. 새벽에 맨날 깬다. 통잠이 최종 목표** | 승리 = 한 번 재우기 ❌ / **통잠(밤새 안 깨고 자기)** ⭕ | **충돌.** 현재는 "밤 종료 시 깊은 잠"이 목표. 새벽 각성·재입면 루프가 핵심이어야 함 |
| G2 | 백일까지만? **백일 후 수면퇴행·원더윅스·이앓이** 있음 | 백일이 끝이 아님. 이후 난이도 스파이크 구간 존재 | **충돌.** 현재 3밤(1일/2일/백일)로 고정(CLAUDE.md 스코프락) |
| G3 | 판단할 때 **시간제한** 있는지? | 실시간 압박(우는 아기 = 초읽기) | 현재 없음(턴제). 도입 시 긴장감↑ |
| G4 | **울면 기저귀부터 확인**이 습관. 안 하고 딴 거 하면 **오판** | "원인 진단" 레이어가 핵심 육아 스킬 | 현재 없음. (이전 CheckDiaper 논쟁의 실제 근거 = 도메인 진실) |

## 1. 원인 진단 레이어 (책 + 언니 → 신규 핵심 메커니즘 후보)

울음/보챔의 **원인을 신호로 읽고 맞히는** 층. 책의 신호 표가 그대로 데이터가 됨.

- **D1 기저귀 우선 확인** (언니): 울면 기저귀부터. 확인 없이 다른 행동 → 오판 페널티.
- **D2 배고픔 신호 단계** (책 표1): 초기(입맛 다심·입 오물·손빪) → 배고플 때(가슴 파고듦·수유자세·꼼지락·때림) → **때 놓친 뒤(머리 좌우·울음)**. 즉 **울면 이미 늦음**. 신호 단계로 "배고픔인지" 판단.
- **D3 피곤 신호 단계** (책 표3): 초기(눈썹 붉음·눈 피함·멍) → 피곤(하품·눈비빔·귀당김·신경질·찡그림·안아달라) → 후기(심하게 움·등 활모양·주먹) → 극도. **피곤을 놓치면 과각성**.
- **D4 온도·습도** (언니): 환경 원인. 안 맞으면 보챔.
- **D5 모로반사** (언니): 놀람반사로 깸. 눕히기/속싸개와 연결(백일밤 "등 센서"의 실제 정체).
- **D6 오판 설계** (언니): 원인이 A인데 B를 하면 실패/시간낭비 + 아기 각성↑.

> 설계 함의: 현재 "행동 고르기"(관리형) 위에 **"원인 읽기 → 맞는 행동"**(진단형) 한 겹이 얹힘.
> 이게 이전에 논쟁하던 바로 그 층인데, **도메인적으로 정당**하다.

## 2. 수면 단계 리얼리즘 (책 206p → 기존 엔진과 정합)

책의 수면 단계가 **현재 코드 SleepStage와 거의 1:1**로 맞는다. 이름·근거를 강화할 기회.

| 책 용어 | 설명 | 현재 SleepStage | 반영 |
|---------|------|-----------------|------|
| 기면(drowsy) | 잠들기 직전, 몸 이완, 호흡 불규칙 | `Drowsy` | 그대로 |
| 활동수면 = **REM** | 얕은 잠, **깨기 쉬움**, 눈꺼풀 떨림·표정·소리 | `Shallow` | "지금 눕히면 깸" 구간 |
| 비활동수면 = **NREM** | 깊은 잠, 잘 안 깸, 규칙적 호흡, **팔다리 축 늘어짐 → 내려놓기 좋음** | `Deep` | **눕히기 성공은 여기서** |

- **S1** 언니 "잠 들고 나면 팔에 힘이 빠져" = NREM 도달 신호. → 눕히기 타이밍 UI로 쓸 수 있음(팔 힘 빠짐 = 눕혀도 됨).
- 즉 **눕히기는 REM(선잠)에서 하면 실패, NREM(깊은 잠)에서 해야 성공** — 이미 코드가 그렇게 동작(Sleep≥85 성공률↑). 근거·연출만 붙이면 됨.

## 3. 아이템 (언니)

| # | 피드백 | 현재 상태 | 반영 판단 |
|---|--------|-----------|-----------|
| I1 | **바운서 빼기** (재울 땐 안 씀) → **옆잠베개 / 좁쌀이불**로 교체 | Bouncer는 이미 LEGACY·제거 예정(decision-register) | **정합.** 대체 아이템 확정 |
| I2 | **토닥인형** 추가 | 없음 | 신규 |
| I3 | **새벽수유** + 준비 미니게임(물온도·소독·분유·끓였다 식힘, 준비 안 되면 빡침, 간편기계 존재) | 없음(Feed만 있음) | 신규(큰 콘텐츠) |
| I4 | **쪽쪽이** — "빠졌다" | **이미 있음**(Pacifier, 밤당 3회) | 이미 반영됨(설명 필요) |
| I4b | 쪽쪽이 **유형별 경험**(이솜=잘 씀 / 아수=안 뭄) + 장단점(안 쓰면 나중에 떼기 편함) | 없음 | 신규(아기 유형과 연동) |
| I5 | **통목욕 / 마사지** 아이템(수면 루틴) | 없음 | 신규 |
| I6 | **실제 브랜드 아이템 + 광고 시청 후 획득** (준용: 광고 수익) | 없음 | 수익화(제품 트랙) |

## 7. 이미 정합하거나 이미 있는 것 (중복 반영 방지)

- 쪽쪽이(Pacifier): **이미 구현됨**. 언니가 못 본 것뿐. 유형별 경험(I4b)만 신규.
- 바운서 제거: 언니 의견 = 기존 decision-register와 **일치**. 옆잠베개/좁쌀이불로 확정만 하면 됨.
- 예민도: 기질 시스템으로 **부분 존재**.
- 수면 단계: 책 REM/NREM = 코드 SleepStage와 **정합**. 근거·연출 강화만.
- 눕히기 타이밍: "팔 힘 빠짐(NREM)에서 눕히기" = **이미 코드가 그런 방향**.

## 8. 결정이 필요한 스코프 갈림길 (이게 먼저)

언니 비전 전체(통잠·수면퇴행·원더윅스·쌍둥이·새벽수유 미니게임·마미톡·광고)는
**해커톤 제출용 5분 게임(CLAUDE.md 스코프락)을 훨씬 넘는 '정식 제품' 비전**이다. 둘을 섞으면 또 무너진다.

- **트랙 A — 해커톤 제출작 (마감 있음)**: 지금 스코프에서 *도메인 검증된 소수*만 반영.
  강력 후보: G1(통잠=승리 재정의), G4/D1(기저귀 우선 진단·오판), 수면 단계 REM/NREM 연출, 바운서→옆잠베개.
- **트랙 B — 정식 제품 (백로그)**: 나머지 전부. 마미톡·광고·쌍둥이·새벽수유 미니게임·수면퇴행 등.

→ **다음 액션: 이 게임이 지금 트랙 A(해커톤)인지, 트랙 B로 방향을 튼 건지 확정.**
